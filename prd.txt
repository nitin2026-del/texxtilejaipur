1. Executive Summary
IndiThread is a direct-to-consumer and business-to-business e-commerce platform purpose-built for an Indian clothing exporter to sell apparel directly to international buyers worldwide. The platform addresses the key pain points of cross-border fashion commerce: secure global payments, currency localisation, smooth buyer authentication, and trustworthy logistics tracking.

The platform will enable the exporter to showcase curated Indian textile and clothing collections, accept payments in multiple currencies via industry-standard international payment gateways, and enforce a frictionless authentication flow that is triggered only at the point of purchase — ensuring a browse-first, buy-authenticated experience.

6. User Authentication — Backend Specification
Authentication is intentionally deferred to the point of purchase. Unauthenticated users can browse the full catalogue and add items to a guest cart. When a user clicks 'Buy Now' or 'Proceed to Checkout', the authentication flow is triggered.

6.1 Authentication Trigger Logic
Event
Behaviour
Browse / Search / View Product
No auth required — fully public
Add to Cart
No auth required — guest cart persisted in localStorage / session
Click 'Buy Now' or 'Checkout'
Auth gate shown — modal overlay with Login / Register tabs
Guest Checkout (future phase)
Optional — not in Phase 1

6.2 Registration Flow
Email + Password signup with real-time validation (password strength indicator)
Email OTP verification (6-digit code; valid for 10 minutes) — required before first purchase
Optional: social login via Google OAuth 2.0 and Facebook Login
Profile auto-populated from social provider (name, email, profile picture)
Country of residence captured to enable tax / duty estimate logic

6.3 Login Flow
Email + password with bcrypt hashing (cost factor 12)
'Forgot Password' — secure reset link via email (token expires in 1 hour)
Remember Me — 30-day persistent session via HttpOnly, SameSite=Strict cookie
Failed login lockout — 5 attempts in 15 min triggers CAPTCHA + 30-min lock

6.4 Session & Token Management
Parameter
Specification
Token Type
JWT (JSON Web Token) — RS256 signed
Access Token TTL
15 minutes
Refresh Token TTL
30 days; stored in HttpOnly cookie
Token Storage
Access token in memory (React state); Refresh token in HttpOnly cookie
CSRF Protection
SameSite=Strict cookie + CSRF token header for state-changing requests
Session Revocation
Token blocklist in Redis on logout / password change / suspicious activity

6.5 Security Requirements
All communication over TLS 1.3 (HTTPS enforced via HSTS)
Passwords never stored in plain text — bcrypt with salt
PII encrypted at rest (AES-256) in PostgreSQL
GDPR-compliant: right to erasure, data portability endpoints required
Rate limiting on auth endpoints: 10 req/min per IP via Redis
Audit log: all login events, password resets, suspicious activity stored in append-only log

7. International Payment System — Specification
7.1 Payment Gateway Strategy
A dual-gateway approach is adopted to maximise global coverage and minimise failed transactions:

Gateway
Role & Coverage
Stripe
Primary gateway for USD, EUR, GBP, AUD, CAD — supports 135+ currencies; PCI-DSS Level 1; 3D Secure 2.0; Apple Pay, Google Pay
Razorpay International
Secondary gateway optimised for India-originated transactions; best INR reconciliation; supports UPI for NRI buyers
PayPal
Buyer-trust option — widely recognised globally; used by 435M+ accounts; Buyer Protection increases conversion

7.2 Accepted Payment Methods
Credit / Debit Cards — Visa, Mastercard, Amex, Discover
Digital Wallets — Apple Pay, Google Pay, PayPal
Buy Now Pay Later — Klarna (EU/UK), Afterpay (AU/US) — Phase 2
Bank Transfers — for wholesale orders above USD 2,000 (manual approval flow)
UPI — for NRI buyers via Razorpay International

7.3 Multi-Currency & Pricing
Feature
Detail

7.5 Security & Compliance
PCI-DSS Level 1 compliance via Stripe.js (card data never touches IndiThread servers)
3D Secure 2.0 (SCA) enforced for EU buyers under PSD2
Fraud detection via Stripe Radar — ML-based rules, block suspicious IPs/cards
FEMA compliance — all inward remittances via AD Category I bank; FIRC documentation
RBI LRS limits — buyer-side compliance via gateway T&Cs

14. Glossary
Term
Definition
DGFT
Directorate General of Foreign Trade — Indian government body governing export/import
IEC
Import Export Code — mandatory 10-digit code for Indian exporters/importers
FEMA
Foreign Exchange Management Act — governs foreign exchange transactions in India
FIRC
Foreign Inward Remittance Certificate — proof of foreign currency receipt
JWT
JSON Web Token — compact, self-contained token used for authentication
PCI-DSS
Payment Card Industry Data Security Standard — security standard for card payments
3DS / SCA
3D Secure / Strong Customer Authentication — EU PSD2-mandated payment auth step
DDP
Delivered Duty Paid — seller pays all import duties; reduces buyer friction
LCP
Largest Contentful Paint — Core Web Vitals metric for page load performance
MOQ
Minimum Order Quantity — minimum number of units for wholesale orders
OMS
Order Management System — software to track orders from placement to delivery
FX
Foreign Exchange — conversion between currencies
